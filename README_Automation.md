# Piste d’évolution – Automatisation

Ce dossier est volontairement laissé simple dans cette première version du projet.

## Évolution possible
- Playwright
- Cypress
- Selenium

## Scénarios candidats à automatiser
- connexion valide
- erreur si champ email vide
- erreur si mot de passe vide
- envoi d’une demande de réinitialisation

## Pourquoi l’ajouter plus tard
Le premier objectif du dépôt est de démontrer une **démarche QA structurée**.  
L’automatisation peut venir dans une seconde version.
