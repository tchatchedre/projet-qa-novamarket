# Comment documenter les preuves de test

Tu n’es **pas obligé(e)** de te filmer en live pour démontrer ton projet.

## Ce qui suffit pour un bon projet GitHub
- cas de test structurés
- rapport de bugs clair
- README propre
- captures d’écran si nécessaire

## Ce qui peut être ajouté en bonus
- une courte vidéo Loom (2 à 4 minutes)
- un GIF montrant un comportement observé
- des captures annotées

## Bonne pratique
Quand tu ajoutes une preuve :
1. donne-lui un nom clair
2. relie-la à un ID de test ou à un ID de bug
3. explique en une ligne ce qu’on doit observer

## Exemple de nomenclature
- `TC-LOGIN-003_erreur_champ_vide.png`
- `BUG-002_double_clic_bouton.mp4`

## Conseil
Pour un recruteur débutant QA, **la qualité de la documentation compte plus qu’une vidéo longue**.
