# Projet QA – Mission simulée NovaMarket

> Projet de démonstration pour une reconversion vers le métier de testeur / QA.  
> Ce dépôt simule une mission réelle de test manuel sur le périmètre **connexion / récupération de mot de passe** d'une application e-commerce fictive.

## Objectif du projet

Montrer de manière concrète ma capacité à :

- comprendre un besoin métier
- rédiger des user stories et critères d’acceptation
- concevoir des cas de test manuels
- documenter des anomalies de manière professionnelle
- proposer une stratégie de test simple et réaliste
- présenter des preuves de test dans un format clair

## Contexte de mission

**Client simulé :** NovaMarket  
**Produit :** application e-commerce web  
**Périmètre testé :**
- connexion utilisateur
- validation des champs
- messages d’erreur
- mot de passe oublié
- règles de base d’accessibilité et d’expérience utilisateur

## Comment lire ce projet

1. Lire le contexte de mission.
2. Consulter la stratégie de test.
3. Ouvrir les user stories et critères d’acceptation.
4. Parcourir les cas de test.
5. Lire les anomalies remontées.
6. Vérifier la checklist de régression.

## Important

Ce projet est une **simulation pédagogique** destinée à démontrer une démarche QA.  
Les anomalies listées dans ce dépôt sont des **exemples de constats de test** dans un contexte de mission simulée.

## Comment présenter ce projet à un recruteur

Vous pouvez résumer ce dépôt ainsi :

> « J’ai simulé une mission QA sur le module de connexion d’une application e-commerce : analyse du besoin, stratégie de test, conception de cas de test, remontée d’anomalies et préparation d’une régression. »
