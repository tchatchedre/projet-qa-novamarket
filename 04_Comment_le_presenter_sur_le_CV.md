# 04 – Comment le présenter sur le CV et en entretien

## Sur le CV

**Projet personnel QA – Mission simulée de test manuel (GitHub)**  
Analyse fonctionnelle, rédaction de cas de test, remontée d’anomalies, stratégie de test et préparation d’une campagne de régression sur un module de connexion e-commerce.

## En entretien

### Version courte
« J’ai simulé une mission QA sur un périmètre réaliste de connexion / mot de passe oublié. J’ai structuré le besoin, préparé des cas de test, identifié des anomalies et formalisé une stratégie de test comme en contexte projet. »

### Version détaillée
- J’ai travaillé sur un périmètre métier simple mais crédible.
- J’ai priorisé les risques avant d’écrire les cas de test.
- J’ai produit des livrables qu’un recruteur peut parcourir rapidement.
- J’ai choisi un format GitHub pour rendre mon travail visible et concret.

## Ce qu’il faut insister

- votre logique de test
- votre rigueur documentaire
- votre compréhension du risque
- votre capacité à communiquer clairement une anomalie
